# PTSD — 캐릭터 바이블

> **이 문서의 목적:** 도구(Movie Gen)의 Character 모델은 `name + headshot + looks[faceImage/bodyImage]`만 담는다. **의상 디테일·생성 프롬프트·컨셉 메모를 담을 필드가 없다.** 그 정보가 휘발되면 shot 프롬프트 작성 시 일관성이 무너지므로, 여기에 보존한다.
> - 에셋 파일(headshot/face/body)의 **슬롯 얼개**는 `data/characters/{slug}.yaml`에 있고, **실물 이미지는 디렉터가 웹 앱에서 업로드**한다(에셋은 local-only, `.gitignore`).
> - 이 문서 = 캐릭터의 "왜·무엇을". YAML = 슬롯 구조. 이미지 = 실물.

## 강지훈 (slug: `jihoon`)

- **배역:** 주인공. 한국 남자 고등/재수생(18–19세). 이야기의 의식(意識)이자 해리의 주체.
- **Look: `hoodie`** (현재 유일 — 오프닝/엔딩에서 사용)
  - 후드티 + 책가방을 맨 **기본 의상**.
  - 학원 가는 길의 평상복 결. 부산하고 조급한 등굣길 톤(시나리오 S1 1~2: 후드티 끈, 반쯤 닫힌 가방 지퍼 = 재호출 디테일).
  - 🔲 색/소재 등 세부는 미정 — 첫 생성 결과 보고 확정.

### 컨셉 메모
- S1 오프닝과 엔딩(거울 구조)에서 동일 의상으로 등장 → 두 장면의 시각적 짝을 위해 **hoodie Look 일관성**이 중요.
- 향후 다른 의상(예: 다른 날/장면)이 필요하면 **새 Look 추가**(예: `uniform`)로 분기.

## 윤수아 (slug: `sua`)

- **배역:** 첫사랑. 한국 여자 학생(18–19세). **환상 = 수아라는 인물.** 감정·행동은 100% 살아있는 여자애, 제약은 정보 층위에만(컨셉노트 1-3, 1-4).
- **Look: `cardigan`** (컨셉상 **단 하나뿐인 의상** — 영화 전체에서 이 의상으로 통일)
  - 오트색 가디건
  - 목 없는 단추 있는 셔츠
  - 검은색 교복바지
  - 하얀 양말
  - **빨간색 낡은 스니커즈** ← 핵심 단서

### 컨셉 메모 (LOCK 연결)
- **빨간 낡은 스니커즈 = 영화 전체의 핵심 단서(LOCK).**
  - 클라이맥스(수아의 집): "구르는 빨간 운동화"가 진짜 사고(죽음)의 파편으로 등장(시나리오 3막, 컨셉노트 6번).
  - 엔딩 시퀀스에서 "사진·빨간 운동화 등 발견"으로 해리가 깨지는 단서.
  - → 의상의 스니커즈와 클라이맥스의 운동화는 **반드시 같은 물건**으로 보여야 함. 색·낡은 정도 일관성 필수. (소품으로도 등장하므로 Prop 등록 검토.)
- **결핍 → 매력 번역(컨셉노트 4-4):** "낡은" 스니커즈, 검소한 차림 = 가난·외로움의 함의. 첫 관람엔 수수한 매력, 재관람엔 처지의 증거. 의상 자체가 양방향 단서.
- **얼굴 노출 규칙(컨셉노트/시나리오 LOCK):**
  - **환상 수아**(1막, 곁에 있는 수아 / A루트): **정면 등장** — "고개 듦 = 환상이 응답하는 동작". → face 시트(정면 포함 5분할)는 **환상 수아 기준**으로 생성.
  - **현실의 수아**(2막 '책 주인', 길에 앉은 뒷모습): **끝까지 뒷모습/옆모습, 정면 금지.** 같은 캐릭터·같은 의상이지만 **shot 연출에서** 정면을 쓰지 않는 것. (에셋엔 정면이 있어야 환상 장면을 만들 수 있음 — 모순 아님.)

## 식별자·에셋 규칙 메모

- **slug는 영문 소문자**([a-zA-Z0-9_.-]만 허용 — asset-store `SAFE_NAME`). 한글 표시명을 담을 필드가 도구에 없어, slug가 화면 라벨도 겸함.
- **에셋 슬롯 구조(캐릭터당):** headshot ×1(캐릭터 단위, 의상 무관) + Look당 **face 시트 ×1 + body 시트 ×1**. (face=5분할, body=3분할이 각각 **한 장의 시트**에 담김 — 그 시트째로 영상 생성기에 투입.)
- **업로드 경로**(웹에서 올리면 자동으로 이 자리에 저장):
  - `characters/{slug}/headshot.<ext>`
  - `characters/{slug}/{look}/face.<ext>` (5분할 시트)
  - `characters/{slug}/{look}/body.<ext>` (3분할 시트)
- YAML의 placeholder 경로는 `.png` 기준으로 적어두었으나, 실제 업로드 확장자(jpg 등)에 맞춰 자동 갱신된다.

### 시트 내부 분할 컨벤션 (권장)
> 도구는 시트 1장만 받고 내부 패널 분할은 모른다. 시트 안의 패널 배치를 캐릭터 간 일관되게 유지한다(생성 워크플로우 기준).
- **face 시트 (5분할):** front / left / right / up / down
- **body 시트 (3분할):** front / side / back
- **headshot:** 얼굴 ID 1장(정면, 의상 무관 — 캐릭터 단위 공통).
