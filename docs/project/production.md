# PTSD (가제) — 제작 실무 노트

> 서사 규칙은 `concept.md`, 씬 목록은 `scenario-outline.md`. 이 문서는 생성 엔진 운용과 협업 방식만 담는다.

## 생성 엔진 — 씨댄스 2.0

- 전 씬 씨댄스 2.0. **1회 생성 최대 15초**, Shot duration은 4–15초로 매 Shot 지정 (Movie Gen 도구 모델과 동일).
- **프롬프트 형식: 산문 대본형이 기본** (테스트상 샷 단위 지정보다 잘 나옴). 비트형 병용. **영어 프롬프트 + 한글 완역 필수.**
- **레퍼런스 슬롯:** 캐릭터 시트(남녀 각 2종, 총 4장) + 배경 ref 시트 + (필요시) 이전 영상.
- **연결(chaining):** 이어지는 씬은 이전 영상을 물려 "이어지는 씬"으로 생성 — 잘 동작. (도구의 prevShotRef/starred Take chaining과 대응.)

## 씨댄스 환각 — 알려진 함정

- 텍스트로 인물 자세/공간을 지시하면 환각 발생 (POV 무릎베개가 소파·양반다리로 깨진 사례). **POV보다 객관 측면샷이 안전.**
- 까다로운 공간 배치: 톱다운 블로킹 다이어그램을 "레이아웃 전용(앵글 복사·라벨 렌더 금지)"으로 넣고 카메라는 텍스트로 오버라이드. 그래도 안 되면 배치 이미지 + Start frame.
- 장소 이동(서울 벤치→인터라켄 벤치)은 **연속 무빙 금지, 매치컷** — 두 클립 따로 생성해 편집에서 매치.
- 3인 이상 + 격앙 연기 조합 위험 — 군중/친구 리액션은 별도 클립으로 분리.

## 배경 미술 방향 [확정 — 2026-06-12]

- **세계 = 2026년 서울 노원구 학군지.** 중계동 은행사거리식 학원가 + 대단지 아파트. 현대적·깔끔·정비됨·밝음. 학원 빌딩/간판, 현대식 가로시설(유리 버스셸터, LED 도착표시 등).
- **금지:** 달동네/빈민가, 손그림 간판·"대기업 금지"식 향수 톤, 낡고 허름한 질감 — **서사상 의도된 대비는 예외.**
- **의도된 예외(낡음 유지):**
  - 지훈(신축 대단지) vs 수아(구축 복도식, 80~90년대 stock) **대비** — 단 "허름한 빈민가"가 아니라 "관리된 구축". 결핍은 직접 주제 아님(concept 7-4).
  - 달동네 언덕 벤치 → **불암산/수락산 자락 근린공원**(정비된, 도시 조망)으로 이전. 인터라켄 벤치 매치컷은 이 근린공원 벤치와 짝.
  - 우체통은 정비된 공원 안의 **오래된 빈티지 디테일 하나**로 유지(감정 단서).
- 전 location YAML·shot 프롬프트에 공통 문구 박음: `2026 Seoul, Nowon-gu academy district — modern, clean, well-maintained, bright.`

## 캐릭터 프롬프트 규칙 [확정 — 가장 중요]

- **영상(shot) 프롬프트에 캐릭터 외형·의상을 절대 쓰지 않는다.** 이름(Jihoon/Sua)과 행동·표정·카메라·배경만 기술. 외형/의상/머리/나이는 **전부 character reference 이미지가 정한다.**
  - _Why:_ ref에 상황별 의상·모습이 이미 정확히 반영돼 있는데 프롬프트로 또 설명하면 씨댄스가 ref를 그대로 안 쓰고 수정해버려 일관성이 깨진다.
  - 어느 look인지(예: sua/accident-day vs sua/uniform)는 **characterRefs 필드**가 지정 — 프롬프트엔 look 이름·"교복 차림" 같은 표현도 쓰지 않는다.
  - 전 프롬프트 공통 지시문: *"Characters appear exactly as in the provided character reference images — do not restyle, re-dress, or alter their appearance."*
  - 단역(ref 없는 인물, 예: 가벼운 사고의 여자아이)은 최소 표현만. 액자 속 사진·신발 등 **set dressing/소품은 로케이션 ref가 담당**(캐릭터 묘사 아님).
  - **외형/의상 묘사가 허용되는 유일한 곳 = `PROMPTS.md`(캐릭터 ref 시트 생성 프롬프트).**

## 핵심 제작 원칙

- **책은 실사 촬영** (소품 제작). AI는 문자에 약함 — 실제 책에 실제 밑줄·메모. AI 생성 화면들 사이 유일하게 만져지는 현실의 닻, 주제적으로도 맞음(환상 속 유일한 현실).
- **함정 재생성 패턴:** 같은 블로킹·같은 대사를 ref만 바꿔 재생성 (#충돌의 배우 교체 공개, 바람 실행 쌍의 혼자/둘 버전). AI 영화의 고유 강점 — 적극 활용.
- 카메라가 또렷이 잡는 모든 글자는 감독 자작 (저작권 — concept 8-1).

## 역할 분담

- **감독:** 기술 운용 전체 + 캐릭터 일관성. (엔진 선택·기술 판단에 Claude 관여 안 함.)
- **Claude:** 시나리오(씬→대본) + 로케 배경 + 서사 책사.
- **책사 역할 (감독 명시 요청):** 검증·동의가 아니라 **비판적 마찰.** 쉽게 동의하지 말고, 회피 패턴을 명명하고, 핵심 미결 질문으로 되돌아오고, 결정의 파급을 비춘다. 감독은 제안을 충돌 재료로 써서 자기 결론에 도달한다. 동의 기계가 되는 순간 책사로서 실패다.
- 감독이 항목을 바꾸려 하면 "전에 확정했잖아"로 막지 말 것. 단 변경이 다른 결정과 충돌하면 충돌을 짚는 것이 책사의 일 (막는 게 아니라 파급을 비추는 것).
